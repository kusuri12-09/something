## H2 Database란?

**H2**는 Java로 작성된 경량 관계형 데이터베이스(RDBMS)다. 개발 및 테스트 환경에서 많이 사용된다.

인메모리와 파일 모드 2가지를 지원한다.

---

## 특징

### 1. 별도 설치 없이 사용 가능

Spring Boot 의존성만 추가하면 바로 사용 가능하다.

```gradle
runtimeOnly 'com.h2database:h2'
```

DB 서버를 따로 실행할 필요가 없다.

---

### 2. 메모리 모드 지원

애플리케이션이 실행되는 동안만 데이터가 존재한다.

```properties
spring.datasource.url=jdbc:h2:mem:testdb
```

장점

* 빠름
* 설정 간단
* 테스트에 적합

단점

* 서버 종료 시 데이터 삭제

---

### 3. 파일 모드 지원

데이터를 파일로 저장할 수 있다.

```properties
spring.datasource.url=jdbc:h2:file:./data/testdb
```

장점

* 재실행해도 데이터 유지

단점

* 실제 운영 DB와 차이 존재

---

### 4. 웹 콘솔 제공

브라우저에서 DB를 확인할 수 있다.

```properties
spring.h2.console.enabled=true
```

접속 주소

```text
http://localhost:8080/h2-console
```

SQL 실행 및 테이블 조회 가능.

---

### 5. SQL 표준 지원

기본적인 SQL 문법을 대부분 지원한다.

```sql
SELECT *
FROM users
WHERE age >= 20;
```

---

### 6. Spring Boot와 호환성 좋음

JPA 실습 시 가장 많이 사용된다.

```java
@Entity
public class User {
    @Id
    @GeneratedValue
    private Long id;
}
```

실행하면 테이블 자동 생성 가능.

---

## 장점

| 장점      | 설명              |
| ------- | --------------- |
| 가벼움     | 별도 서버 필요 없음     |
| 빠름      | 메모리 모드 지원       |
| 설정 간단   | 의존성 추가만으로 사용 가능 |
| 테스트 친화적 | 단위 테스트, 실습에 적합  |
| 웹 콘솔    | DB 상태 확인 가능     |

---

## 단점

| 단점                      | 설명                   |
| ----------------------- | -------------------- |
| 실무 사용 거의 없음             | 운영 DB로 잘 사용하지 않음     |
| PostgreSQL/MySQL과 차이 존재 | SQL 문법 및 동작이 다를 수 있음 |
| 성능 검증 불가                | 실제 운영 환경과 다름         |
| DB 특화 기능 부족             | 파티셔닝, 복제 등 없음        |

---

## MySQL/PostgreSQL과 비교

| 항목     | H2          | MySQL/PostgreSQL |
| ------ | ----------- |------------------|
| 설치     | 불필요         | 필요               |
| 실행     | 애플리케이션 내부   | 별도 서버            |
| 속도     | 매우 빠름       | 상대적으로 느림         |
| 데이터 보존 | 메모리 모드 시 불가 | 가능               |
| 운영 환경  | 거의 사용 안 함   | 많이 사용            |
| 테스트    | 매우 적합       | 가능하지만 무거움        |
